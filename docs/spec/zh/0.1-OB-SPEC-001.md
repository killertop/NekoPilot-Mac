# OneBox 标准行为规范

**文件编号：** OB-SPEC-001
**版本：** 1.0.0
**发布日期：** 2026-03-10
**状态：**  草稿

---

## 前言

本规范规定了 OneBox 客户端与服务端之间远程配置文件交互的标准行为，旨在确保互操作性与数据安全性。

本文件中，关键词"**必须**"、"**不得**"、"**应当**"、"**不应**"、"**建议**"、"**不建议**"、"**可以**"的含义与 [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119) 中的定义一致。

---

## 1. 范围

本规范适用于所有实现 OneBox 协议的客户端与服务端。OneBox **应当** 兼容 sing-box 官方客户端的订阅链接格式，并在其基础上附加本规范所定义的扩展要求。

---

## 2. 术语与定义

### 2.1 远程配置文件（Remote Configuration File）

市场上通常称为"订阅链接"的资源，其本质为托管于远程服务器上的配置文件。本规范统一采用**远程配置文件**这一术语。

### 2.2 远程配置链接（Remote Configuration URL）

用于获取远程配置文件的 URL。

### 2.3 元信息（Metadata）

服务端通过 HTTP 响应头向客户端传递的附加信息。

### 2.4 客户端（Client）

实现 OneBox 协议的终端应用程序。

### 2.5 服务端（Server）

提供远程配置文件及相关元信息的 HTTP 服务提供方。

---

## 3. 远程配置链接规范

### 3.1 传输安全

远程配置链接**必须**以 `https://` 为前缀，以确保数据传输过程中的机密性与完整性。使用非加密协议（如 `http://`）的链接**不得**被客户端信任或使用。

### 3.2 HEAD 请求支持

服务端**应当**支持对远程配置链接的 `HEAD` 请求，以允许客户端在不下载配置文件主体的前提下获取元信息。

---

## 4. 元信息规范（HTTP 响应头）

元信息通过 HTTP 响应头返回，适用于 `HEAD` 请求及 `GET` 请求的响应。

### 4.1 `subscription-userinfo`（必填）

服务端**必须**在响应头中包含 `subscription-userinfo` 字段，用于传递用户的流量使用情况及订阅有效期。

**格式：**

```
subscription-userinfo: upload=<bytes>; download=<bytes>; total=<bytes>; expire=<unix-timestamp>
```

**示例：**

```
subscription-userinfo: upload=0; download=58808587680; total=1099511627776; expire=4889654302
```

**字段定义：**

| 字段 | 数据类型 | 必填 | 说明 |
|------|----------|------|------|
| `upload` | 非负整数（字节） | 必填 | 当前计费周期内的已上传流量 |
| `download` | 非负整数（字节） | 必填 | 当前计费周期内的已下载流量 |
| `total` | 正整数（字节） | 必填 | 当前计费周期内的总流量配额 |
| `expire` | 整数（Unix 时间戳，秒，无小数部分） | 必填 | 订阅到期的 Unix 时间戳 |

---



### 4.3 `rewrite-config-url`（可选）

服务端**可以**在响应头中包含 `rewrite-config-url` 字段，用于向客户端提供规则配置元信息的 JSON 文件链接。

详细的 rewrite-config 规范请参见专用文档：[OB-REWRITE-CONFIG-SPEC](./0.2-OB-REWRITE-CONFIG-SPEC-001.md)


---

## 5. 客户端标识规范（User-Agent）

### 5.1 UA 格式

客户端在发起请求时，`User-Agent` 请求头**必须**符合以下格式：

```
<ClientName>/<Version> (<OS> <Architecture> <OSVersion>; sing-box <SingBoxVersion>; language <Language>)
```

**示例：**

```
SFI/1.0.5 (ios arm64e 26.3.1; sing-box 1.13.0; language zh-Hans-CN)
SFM/1.3.6 (macos aarch64 26.3.1; sing-box 1.13.0; language zh-Hans-CN)
```

---

### 5.2 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ClientName` | 字符串 | 必填 | 客户端标识符，取值见第 5.3 节 |
| `Version` | 语义化版本号 | 必填 | 客户端应用版本号。版本号 ≥ 1.11 通常标识 sing-box 官方客户端；< 1.11 通常标识 OneBox 客户端，但不排除其他第三方实现 |
| `OS` | 字符串 | 必填 | 操作系统标识，如 `ios`、`macos`、`android`、`windows`、`linux` |
| `Architecture` | 字符串 | 必填 | 处理器架构标识，如 `arm64e`、`aarch64`、`x86_64` |
| `OSVersion` | 字符串 | 必填 | 操作系统版本号，如 `26.3.1` |
| `SingBoxVersion` | 语义化版本号 | **必填** | sing-box 内核版本号，如 `1.13.0`。服务端**必须**依据此版本提供对应的远程配置文件 |
| `Language` | BCP 47 语言标签 | 必填 | 客户端界面语言，如 `zh-Hans-CN` |

---

### 5.3 客户端标识符 (ClientName)

| 标识符 | 平台 | 适用客户端情况 |
|:---:|---|---|
| `SFI` | iOS (iPhone / iPad) | 官方及 OneBoxM 客户端使用 |
| `SFM` | macOS | 官方及 OneBox 客户端使用 |
| `SFA` | Android | 官方及 OneBoxM 客户端使用 |
| `SFT` | Apple TV | 仅官方客户端支持，OneBox 暂无 |
| `SFW` | Windows | OneBox 客户端专有 |
| `SFL` | Linux | OneBox 客户端专有 |

---

## 6. 深度链接规范 (Deep Linking)

### 6.1 URI 方案

客户端**必须**注册并支持 `oneoh-networktools://` URI 方案，用于实现跨应用或从外部浏览器触发的快速配置导入。

### 6.2 配置导入指令

用于导入远程配置文件的深度链接**必须**符合以下格式：

```
oneoh-networktools://config?data=<Base64-Encoded-URL>[&apply=1]
```

### 6.3 参数定义

*   **`data`**: **必填**。该参数的值**必须**是经过 Base64 编码（符合 [RFC 4648](https://www.rfc-editor.org/rfc/rfc4648) 标准）的远程配置链接（Remote Configuration URL）。
*   **解码行为**: 客户端在接收到请求时，**必须**先对 `data` 参数进行 Base64 解码，获取原始的 URL，并随后按照本规范第 3 节的要求进行后续处理。
*   **`apply`**: **可选**。当该参数值为 `1` 时，客户端**必须**在成功导入配置后，立即使用该配置启动服务。若服务当前正在运行，客户端**必须**先中断当前运行，再使用新配置重新启动。

**示例：**

若原始远程配置链接为 `https://example.com/config.json`，其 Base64 编码后的值为 `aHR0cHM6Ly9leGFtcGxlLmNvbS9jb25maWcuanNvbg==`。

对应的深度链接为：
`oneoh-networktools://config?data=aHR0cHM6Ly9leGFtcGxlLmNvbS9jb25maWcuanNvbg==`

若需导入后立即应用并启动，对应的深度链接为：
`oneoh-networktools://config?data=aHR0cHM6Ly9leGFtcGxlLmNvbS9jb25maWcuanNvbg==&apply=1`
