
# OneBox rewrite-config 规范

**文件编号：** OB-REWRITE-CONFIG-SPEC  
**版本：** 1.0.0  
**发布日期：** 2026-03-11  
**状态：** 草稿

---

## 前言

本规范定义了 OneBox 客户端与服务端之间通过 `rewrite-config-url` 进行规则配置元信息交互的标准行为，旨在确保互操作性、数据安全性与系统可扩展性。

本文件中，关键词"**必须**"、"**不得**"、"**应当**"、"**不应**"、"**建议**"、"**不建议**"、"**可以**"的含义与 [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119) 中的定义一致。

## 1. 范围

本规范适用于所有实现 OneBox 协议的客户端与服务端。

## 2. 术语与定义

### 2.1 重写配置链接 (rewrite-config-url)

服务端通过 HTTP 响应头下发的 URL，指向包含规则配置元信息的 JSON 资源。

### 2.2 重写配置授权标识 (rewrite-config-auth)

服务端下发的自定义授权凭据（如 Token 或 UUID），客户端在访问 `rewrite-config-url` 时**必须**将其包含在请求头中。

---

## 3. 响应头字段规范

服务端在响应远程配置链接（见 OB-SPEC-001）的请求时，**可以**在 HTTP 响应头中包含以下字段：

### 3.1 `rewrite-config-url`

*   **要求**：**必须**是以 `https://` 为前缀的有效 URL，以确保元信息传输的安全。
*   **用途**：指定规则配置元信息 JSON 文件的位置。

### 3.2 `rewrite-config-auth`
*   **要求**：由服务端下发的自定义授权标识。
*   **特殊情况**：当服务端未下发此字段，或该字段值为空时，客户端对应的访问行为见第 4 节。

**示例：**
```http
rewrite-config-url: https://example.com/rules-meta.json
rewrite-config-auth: 2f8c1e7a-1b2c-4d3e-9f0a-123456789abc
```

---

## 4. 授权访问机制 (rewrite-config-auth)

本节详细规定了客户端访问 `rewrite-config-url` 资源时的身份验证行为。

### 4.1 请求头构造规则

客户端在发起针对 `rewrite-config-url` 的 HTTP 请求时，**必须**将 `rewrite-config-auth` 字段名作为 HTTP 请求头的键（Key）：

1.  **非空值情况**：若服务端下发的 `rewrite-config-auth` 包含有效字符串，客户端在发起请求时，**必须**设置一个名为 `rewrite-config-auth` 的 HTTP 请求头，其值（Value）为服务端提供的该字符串。
2.  **空值或缺省情况**：若服务端未下发 `rewrite-config-auth` 或其值为空，客户端在发起请求时，仍**必须**设置名为 `rewrite-config-auth` 的 HTTP 请求头，其值（Value）设置为 `empty`。

### 4.2 传输安全要求

客户端**必须**仅在支持 TLS 的加密连接（HTTPS）上发送此类包含授权标识的请求。

---

## 5. 客户端访问行为

### 5.1 内核版本适配
1.  **User-Agent 携带**：客户端在请求 `rewrite-config-url` 时，**必须**携带符合 [OB-SPEC-001](./0.1-OB-SPEC-001.md) 第 5 节规范的 `User-Agent` 请求头。
2.  **服务端动态响应**：服务端**应当**解析 UA 中的 `SingBoxVersion` 字段，并返回与之匹配的 JSON 资源内容。

---

## 6. JSON 资源结构规范

重写配置元信息文件**必须**为符合形式的 JSON 格式资源。

### 5.1 标准字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `tun-rules-url` | 字符串 (URL) | 可选。指向 TUN 模式规则配置模板（JSON/C 格式）的链接。 |
| `mixed-rules-url` | 字符串 (URL) | 可选。指向混合模式规则配置模板的链接。移动端客户端**应当**忽略此字段并回退至 `tun-rules-url`。 |

**示例结构：**
```json
{
  "tun-rules-url": "https://cdn.example.com/conf/1.13.0/zh-cn/tun-rules.jsonc",
  "mixed-rules-url": "https://cdn.example.com/conf/1.13.0/zh-cn/mixed-rules.jsonc"
}
```

---

## 7. 异常处理与服务端响应

### 7.1 授权验证失败
若 `rewrite-config-auth` 验证失败，服务端**应当**返回 `403 Forbidden` 状态码。

### 7.2 客户端容错
客户端在处理 `rewrite-config-url` 请求时：
1.  **应当**仅对 HTTP `200 OK` 响应进行解析。
2.  若收到 `403`、`404` 或其他非 `2xx` 状态码，客户端**必须**忽略该元信息配置，并按照默认流程或此前缓存的有效配置继续执行，不应中断主程序的连接逻辑。
