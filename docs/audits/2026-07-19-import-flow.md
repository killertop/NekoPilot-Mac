# 本地节点导入与连接流程审计

日期：2026-07-19

## 覆盖流程

1. 首次启动，进入“配置”空状态。
2. 打开“添加配置”，阅读订阅与单节点导入说明。
3. 导入 VLESS、Trojan、VMess 或 Shadowsocks 单节点。
4. 导入完成后返回主页并连接。
5. 处理失效链接、缺失服务器、缺失认证信息，以及缺少 Reality `pbk` 的节点。

## 已执行改进

- 手动导入的配置现在会自动成为当前配置，但不会自动启动代理。
- 进度弹窗明确显示“已设为当前配置，可在主页连接”，不再把未发生的启动动作显示为完成。
- Rust 导入器新增 Shadowsocks SIP002 单节点；本地节点总共支持 VLESS、Trojan、VMess、Shadowsocks。
- 导入错误按原因展示：服务器/端口、认证、Reality 公钥、无可用节点或暂不支持的协议。
- 选择器将本地节点标识为“不自动更新”，不再误称为本地文件。
- 空状态去除了遗留 OneBox 文案；输入框采用短提示和独立协议说明，避免 macOS 窄窗口截断。
- SQLite 配置为 WAL、5 秒忙等和 NORMAL 同步策略，降低导入与选择并发时的锁冲突，同时不增加后台轮询。

## 可视化检查

- 空状态：通过；文案准确说明可导入订阅或单节点。
- 添加弹窗：通过；协议说明完整且未被截断。
- 已选中后连接：代码路径已确保选择存储在引擎启动前完成；未用真实代理凭据触发连接，避免改变现有网络路由。

## 验证

- TypeScript：`tsc --noEmit` 通过。
- 前端测试：7 个文件、39 个测试通过。
- Rust 导入模块：6 个测试通过，覆盖 VLESS Reality、Trojan、VMess、Shadowsocks 与事务。
- Rust Clippy：`-D warnings` 通过。
- macOS `.app`：完整打包与严格代码签名校验通过。

## 后续优先级

2. 增加“从剪贴板粘贴”入口，并在读取前明确说明权限与内容范围。
3. 对真实节点增加可选的“测试连通性”按钮，只在用户点击后发起网络请求。

## 性能回访

- 节点下拉原先会为每一个已挂载行自动发起一次 `/delay` 请求；节点较多时会让 WebView、Clash API 和逐项动画同时繁忙。
- 现改为仅测试当前节点；切换节点后才测试新节点。非当前行保留稳定的占位宽度，不会造成列表跳动。
- 下拉列表移除逐行入场动画，页面切换缩短为 160 ms 的纯淡入，减少 WebKit 合成层和布局工作。
- 连接确认轮询支持取消，离开/切换状态后不会继续等待并更新已卸载页面。
- 空闲态采样：WebContent 物理占用约 38 MB、CPU 0%，未发现持续忙等；问题主要在大节点列表的交互峰值，而非空闲后台负载。
