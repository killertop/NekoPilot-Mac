# NekoPilot Mac 视觉方案审计

日期：2026-07-19

## 结论

当前方案的 Apple system color、浅色卡片、连接主按钮和 grouped list 已经形成了可识别的视觉基线，不建议推倒重做颜色或动画。

更好的重构方向是先统一产品骨架：明确它是“macOS 紧凑代理面板”，还是“移动端式卡片应用”。以当前 371×600 的窗口尺寸看，推荐保留紧凑面板形态，重构首页的连接上下文、配置项信息层级和页面导航；如果准备面向高频多配置管理，再考虑放大窗口并改为侧栏工作区。

## 当前运行截图

1. 首页：主连接按钮突出，当前配置和节点选择可见；但节点选择在未连接时像一个失效输入框，连接目标不够明确。
2. 配置管理：列表和操作卡片结构清楚；节点标题在窄窗口内被截断，协议和本地节点身份不够有层次。
3. 添加配置：模态框轻量、操作简单；“订阅链接或节点链接”和协议说明的层级仍偏弱。
4. 设置：辅助功能树已切换到设置内容，但截图在当前运行包中保持空白，且底部仍残留旧 tab 高亮。
5. 规则：辅助功能树已切换到规则内容，但截图同样保持空白；需要先确认是否为真实渲染问题。

## 关键发现

### P0：先核查页面切换后的空白渲染

从配置页切到设置、规则，再回到首页时，辅助功能树显示了新页面内容，但可见截图出现空白，底部 tab 高亮也出现残留。若在真实用户窗口可复现，这是发布阻断问题；若只发生在自动截图层，也应补一轮人工窗口切换确认。

### P1：平台语言在 macOS 与 iOS 之间摇摆

底部四 tab、圆角 grouped card、HomeKit 式大电源 tile 都很像 iOS；但窗口有 macOS 标题栏、固定 371×600 尺寸，配置页又承担桌面工具的多配置管理。单个组件都好看，组合后平台身份不够确定。

### P1：首页应围绕“当前连接对象”重排

现在的视觉顺序是：电源 tile → 状态 → 规则/全局 → 当前配置 → 节点选择。用户真正需要确认的是“我要连哪个配置里的哪个节点”。建议将配置、节点、连接状态合成一个连接上下文卡片；保留大电源按钮，但让卡片直接显示当前配置、节点、协议/本地节点标签和连接执行状态。

### P1：配置列表需要减少截断和视觉噪声

当前标题把协议、节点名、地址挤在同一行，窄窗口下会变成省略号。建议拆成“名称主行 + 协议/本地节点标签 + 主机信息次行”，把“不自动更新”做成明确的低强调标签，而不是依赖一行小字解释。

### P1：操作主次还可以更明确

“更新全部”和“添加配置”现在都是蓝色全宽行，权重接近；对于空配置或需要继续添加节点的场景，“添加配置”应是主动作，“更新全部”应是次动作，并在本地节点存在时明确说明它不会参与更新。

### P2：可读性和键盘焦点需要复核

11px section label、tertiary label 和底部未选中 tab 的对比度偏低；截图中自动化点击后的 tab 出现很大的蓝色焦点框，也需要设计成明确但不破坏布局的键盘焦点样式。不能只依赖颜色表达“未连接 / 禁用 / 本地节点”。

## 推荐重构边界

### 方案 A：紧凑 macOS 面板（推荐）

- 保持当前窗口尺寸和 Apple system color。
- 把首页改成“连接上下文卡片 + 电源按钮 + 轻量模式切换”。
- 配置页统一为“配置名称、来源类型、协议标签、更新能力、更多操作”。
- 设置页加入 General / Network / Advanced 分组标题，避免一整页无语义分区。
- 统一字号、分隔线、禁用态、焦点态和 modal 宽度。

这是视觉和结构的重构，不需要重写代理业务逻辑。

### 方案 B：桌面工作区

如果用户经常管理很多订阅、规则和节点，可以把窗口放大并允许调整尺寸，采用左侧导航 + 右侧内容区。这个方向会更像真正的 macOS 工具，但会改变产品占屏方式，应该在确认使用场景后再做。

## 当前证据边界

本轮已保存并检查首页、配置管理、添加配置的当前运行截图；设置和规则的截图记录了空白渲染现象。未覆盖暗色模式、连接成功/失败/测试中状态、长订阅列表、键盘导航和真实网络错误文案，这些需要下一轮单独验证。
